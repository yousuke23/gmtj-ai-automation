# デプロイ — セキュリティ原則（社内MVP）

## 必須

- **`.env` / `.htpasswd` / API キーは git に含めない**（`.gitignore` 済み）。  
- インターネットに晒す前に **VPN / Basic 認証 / IP 制限**のいずれか（複数併用推奨）。  
- Basic 認証のみでは **HTTPS なしの場合パスワードが漏えいしやすい** → 可能なら **TLS 終端**（`deploy/caddy/` の例）または VPN 内のみ。  
- ログに **パスポート・カード・正確な住所**を残さない（`kb/policy.md` と整合）。

## Open WebUI

- 管理アカウントのパスワードをデフォルトのままにしない。  
- モデル API キーは **環境変数**で渡し、画面キャプチャに写らない運用にする。

## nginx（auth-stack）

- `deploy/nginx/.htpasswd` は **強いパスワード**＋ファイル権限 `600` を推奨。  
- 不要になったらコンテナを止め、ポートを開放しない。

## 社外公開する場合（インターネットに URL を出す）

**自宅 Mac の `127.0.0.1:8080` のままでは社外から開けない。** 次のいずれかで「届く入口」を用意する。

1. **推奨（運用が楽）:** VPS や社内サーバーに Docker を置き、**TLS 終端**（`deploy/caddy/README.md` の本番ドメイン手順）＋ **Basic 認証**（`deploy/nginx/README.md`＋`docker-compose.auth-stack.yml`）を **443 で公開**。DNS でドメインをそのサーバーへ向ける。  
2. **トンネル:** Cloudflare Tunnel 等でローカルへトンネルする場合も、**TLS と入口の認証**を必ずかける（トンネル URL をそのまま共有しない運用を推奨）。  
3. **パートナー限定の「社外」**が目的なら、**VPN（Tailscale 等）＋既存の 127.0.0.1** だけにして、**インターネット全体にポートを開けない**選択肢がある。

**必須チェック:** `SERVICE-LAUNCH.md` の **フェーズ4**（人間承認）・本ファイルの **VPN / Basic / IP 制限**・Open WebUI の **強い管理者パスワード**・**API キーを画面に出さない**。

## 全世界向け（一般インターネットに URL を出す）

**脅威が最大**（ボットのスキャン・ブルートフォース・スクレイピング・DDoS）。次を満たさない限り、**公開しない方が安全**。

- **TLS 必須:** 独自ドメイン＋**Let's Encrypt 等の本番証明書**（`deploy/caddy/README.md` の本番向け記述を実ドメインに合わせる）。HTTP のみで公開しない。  
- **入口の認証を二段以上検討:** nginx の **Basic** に加え、**Open WebUI 側のログイン**（強いパスワード／公式が案内する **OAuth・SSO** が使えるなら優先）。**管理画面と一般利用を同一 URL で無防備に晒さない**運用を検討。  
- **ホスト:** 自宅 Mac直公開は避け、**クラウド VPS またはマネージド基盤**に載せ替える。ファイアウォールで **必要最小のポートだけ**開ける（例: SSH **22**、HTTP **80**、HTTPS **443**）。不要なサービスは閉じる。  
- **レート制限・WAF:** リバースプロキシ前段（Cloudflare、nginx `limit_req` 等）で **ブルートフォース対策**。  
- **ナレッジ:** `kb/policy.md` に沿い、**個人情報・未公開契約・社内秘**を Knowledge に載せない。公開 URL では **デモ用ナレッジ**に絞る選択肢がある。  
- **モデル API キー:** サーバー環境変数のみ。キー漏洩時の **即ローテ**手順を決めておく。  
- **法務・表記:** 全世界向けなら **利用規約・プライバシー・苦情窓口**（サイト上のリンク）を別途用意する（本リポには書かない）。  
- **監視:** 障害・不正検知の連絡先（オンコール）を決めてから公開する。  
- **具体構成例:** **[WORLDWIDE-STAGING.md](./WORLDWIDE-STAGING.md)** と **`docker-compose.worldwide-staging.yml`**（VPS・DNS・Let's Encrypt 前提）。

- 依存イメージを更新する場合は **変更履歴（CVE）**を確認し、検証環境で起動確認してから本番相当へ。

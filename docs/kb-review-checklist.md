# `kb/` レビュー用チェックリスト（人間向け）

AIコンシェルジュ（14）向けナレッジを更新したあと、**公開・Knowledge 取り込みの前**に目を通す用です。すべてにチェックが付くまでリリースしない運用を推奨します。

## 事実・出典

- [ ] 営業時間・料金・可否を**推測で断定**していない
- [ ] 数値・ルールを書いている場合は **`kb/sources.md` の推奨フォーマット**（出典URL・確認日）になっている、または「公式で確認」と明記している
- [ ] 他社・他施設の**根拠のない優劣**がない

## 安全・法務・個人情報

- [ ] 医療・法律・ビザ・投資の**個別判断**に踏み込んでおらず、エスカレーション方針に沿っている（`kb/policy.md`）
- [ ] 個人情報（パスポート・カード番号等）の**収集を促していない**
- [ ] 神社・宗教施設について、**不確実な慣習をでっち上げ**ていない

## 多言語・表記

- [ ] 英語・日本語で**同じ意図の方針が矛盾**していない（必要なら韓国語 `.ko.md` も）
- [ ] 地名・施設名は `kb/area-atami-izu/placenames.md` や `kb/brand/` と**表記が食い違っていない**

## テンプレ整合（意図を増やしたとき）

- [ ] `kb/intents.yaml` に `id` を足したら **`kb/templates/{id}.ja.md` / `.en.md`** を同じ変更で追加した
- [ ] 韓国語チャネルがあるなら **`{id}.ko.md`** も追加した
- [ ] 可能なら **`eval/golden-questions.md`** に評価用の質問を1問以上足した
- [ ] 採点基準を変えたら **`eval/scoring-rubric.md`** を更新した
- [ ] `bash scripts/verify-kb-templates.sh` が **成功**する

## デプロイ後（Open WebUI を使う場合）

- [ ] Knowledge の再取り込み／再インデックスが必要なら実施した（`deploy/openwebui-knowledge.md`）
- [ ] 社内方針どおり **VPN / 認証 / TLS** のいずれかでアクセスを制限している

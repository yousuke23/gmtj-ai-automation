Automation · SNS · LINE

SNS自動投稿 & 公式LINE 基盤

ブログ記事公開を起点に、TikTok / Instagram / YouTube / X / Facebook 向けのキャプション・ハッシュタグ・画像URLを一括生成し、 スケジュール投稿キューと外部オーケストレーション（Make / Zapier 等）へ渡せます。AI TARNAR Voice School と Izu Music Fund を最優先ブランドとしてプリセット済みです。

連動フロー（シンプル運用）
  1. 記事を site/tarnar/blog/ または site/izu-fund/blog/ に公開（既存ビルドスクリプト可）
  2. このページで「プレビュー生成」→ 各SNS用テキストをコピー、または「公開パイプライン」でサーバー側一括生成
  3. 環境変数 AUTOMATION_BRIDGE_URL に Make/Zapier のWebhookを設定 → Meta Graph / X API v2 / TikTok Content 等へ分岐
  4. 任意: LINE_*_NOTIFY_USER_ID 登録済みユーザーへ新着記事プッシュ（社内通知向け）
  5. 1日複数回: site/automation/queue.json に予約行を追加しデプロイ → Netlify スケジュール関数 social-daily-tick が期限到来分を bridge へPOST（UTC 02:08:14:20 付近＋補正）

生成コンソール

同一オリジン上の Netlify Function build-social-kit を呼び出します。

本番テスト用サンプル（voice-guide-1501。URL はデプロイ後の本番ドメインに合わせてください）

「公開パイプライン実行」後、ステータス直下に snsPlatformOrder / bridge / lineNotify のJSONが表示されます（ローカル静的プレビューでは Functions は未接続のためエラーになります）。

cURL でパイプラインを叩く（CLIテスト）

OPS_PUBLISH_SECRET と本番オリジンに差し替えて実行します。

curl -sS -X POST "https://gmtj-japan-music-tourism.netlify.app/.netlify/functions/blog-publish-hook" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_OPS_PUBLISH_SECRET" \
  -d '{"brand":"tarnar","article":{"title":"初深海熱水噴出孔採音の16分音符だけの舌位置トラッキング","url":"https://gmtj-japan-music-tourism.netlify.app/tarnar/blog/ja/voice-guide-1501.html","summary":"AI TARNAR Voice School の実践メモ（batch18）。ブログ公開からSNSパッケージ・bridge・LINE通知まで一括テスト用。"}}'
本番テスト手順（チェックリスト）
  1. 「TARNAR（1501）でフォーム投入」または Izu サンプルで、公開済み記事URL・要約をセット（例: voice-guide-1501）
  2. Netlify に OPS_PUBLISH_SECRET を設定し、ローカルと同じ値をこのページに入力
  3. 任意: AUTOMATION_BRIDGE_URL（Make/Zapier）を設定 → パイプライン実行後 bridge に ok が付くか確認
  4. 任意: LINE_*_CHANNEL_TOKEN と LINE_*_NOTIFY_USER_ID → lineNotify.sent を確認
  5. 「プレビュー生成」で build-social-kit のみ検証 → 問題なければ「公開パイプライン実行」
  6. 下のプレビュー欄で各SNSキャプションをコピーし、手動投稿または bridge 先で配信

公式LINE Webhook URL（LINE Developers に登録）

チャネルごとに別URLを設定してください（チャネルシークレット・トークンは Netlify の環境変数）。

  • /.netlify/functions/line-tarnar — LINE_TARNAR_CHANNEL_SECRET / LINE_TARNAR_CHANNEL_TOKEN
  • /.netlify/functions/line-izu-music-fund — LINE_IMF_CHANNEL_SECRET / LINE_IMF_CHANNEL_TOKEN

自動応答キーワード例: ブログ / クーポン / 予約 / メニュー / 友だち / こんにちは（Izu側は「ファンド」「ターナー」も）

ブログ公開 → SNS 一括（サーバー側）

POST /.netlify/functions/blog-publish-hook に Authorization: Bearer OPS_PUBLISH_SECRET と JSON {"brand":"tarnar","article":{"title":"…","url":"https://…/tarnar/blog/ja/….html","summary":"…"}} を送ると、TikTok / Instagram / YouTube / X / Facebook 向けキャプションを socialPack として返します。 AUTOMATION_BRIDGE_URL が設定されていれば同内容を Webhook へ転送し、任意で LINE 社内通知も送れます。

環境変数チェックリスト（Netlify）

  • OPS_PUBLISH_SECRET — 公開パイプライン認証
  • AUTOMATION_BRIDGE_URL — SNS/LINE配信を委譲するWebhook（推奨）
  • SITE_CANONICAL_URL — 本番ドメイン（キューJSON取得に使用）
  • LINE_TARNAR_NOTIFY_USER_ID / LINE_IMF_NOTIFY_USER_ID — 任意の社内通知先LINEユーザーID
  • 既存チャット: ANTHROPIC_API_KEY
